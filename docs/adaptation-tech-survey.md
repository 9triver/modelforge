# 跨区域模型适配 — 核心技术方法梳理

## 问题本质

电力行业 AI 模型跨区域共享的核心障碍：**源域和目标域之间存在分布偏移（Distribution Shift）**。

这种偏移有多个层次：

```text
P(X) 偏移 — 输入特征分布不同（北方冬季用暖气，南方夏季用空调）
P(Y|X) 偏移 — 相同特征下目标变量的响应关系不同（产业结构差异导致同温度下负荷不同）
特征空间不同 — 有些地区根本没有某些特征（如无供暖指数）
```

五类技术分别在不同层次解决这个问题。

---

## 一、特征工程适配

### 1.1 要解决的问题

跨区域差异首先体现在特征空间：不同地区可用的特征不同，同一特征的重要性也不同。

### 1.2 核心技术方法

#### 1. 特征重要性评估

目的：识别在目标场景下哪些特征真正有用。

| 方法 | 原理 | 适用场景 | 参考 |
| --- | --- | --- | --- |
| **SHAP (SHapley Additive exPlanations)** | 博弈论 Shapley 值，度量每个特征对预测的边际贡献 | 任意模型，最准确但计算量大 | Lundberg & Lee, 2017 (NeurIPS) |
| **Permutation Importance** | 随机打乱某特征，观察模型性能下降幅度 | 任意模型，模型无关的评估 | Breiman, 2001 |
| **Tree-based Importance** | 树模型分裂时各特征贡献的信息增益总和 | XGBoost / LightGBM / RF | Friedman, 2001 |
| **Lasso (L1 正则化)** | L1 惩罚自动将不重要特征系数压到 0 | 线性模型，做特征筛选 | Tibshirani, 1996 |
| **Mutual Information** | 度量特征与目标变量之间的互信息量 | 非线性关系也能捕获，不依赖模型 | Kraskov et al., 2004 |

**电力场景选择建议**：

- 树模型（XGBoost 为主）：直接用 Tree Importance + SHAP 组合
- 深度学习模型（LSTM/Transformer）：用 SHAP DeepExplainer 或 Permutation Importance
- 快速筛选：Mutual Information（不需要训练模型）

#### 2. 自动特征选择

目的：从影响因子库中为目标场景自动挑选最优特征子集。

| 方法 | 类型 | 原理 |
| --- | --- | --- |
| **Filter** | 预筛选 | 基于统计量（相关系数、MI、方差）独立评估每个特征 |
| **Wrapper (RFE)** | 迭代剔除 | 反复训练模型，每轮剔除最不重要的特征 |
| **Embedded (Lasso/ElasticNet)** | 内嵌 | 训练过程中自动做特征选择 |
| **Boruta** | 统计检验 | 与随机特征对比，保留显著优于随机的特征 |
| **Sequential Feature Selection** | 贪心搜索 | 逐步添加/删除特征，优化验证集指标 |

**对于跨区域适配的关键问题**：

- 不是简单地选 top-K 特征，而是要发现**地区特有因子**
- 例：北方的"供暖指数"可能在南方不存在，但"台风指标"在南方很关键
- 方法：先用全量因子库训练 → SHAP 分析 → 识别出该地区的关键因子组合

#### 3. 特征空间对齐

目的：当源域和目标域的特征空间不完全重叠时，做对齐或映射。

| 方法 | 原理 |
| --- | --- |
| **特征映射表** | 人工定义字段名到标准特征名的映射（最基础，最实用） |
| **特征填补** | 缺少的特征用代理特征或统计方法补全 |
| **特征投影** | PCA / Autoencoder 将不同特征空间投影到共享低维空间 |

**电力场景实际情况**：因为电力行业的数据格式相对标准化（D5000 规范等），特征映射表 + 缺失特征填补通常就够了，不一定需要复杂的投影方法。

## 二、参数迁移与快速调优

### 2.1 要解决的问题

模型结构不变，通过调整超参数适配新场景。这是成本最低的适配手段。

### 2.2 核心技术方法

#### 1. 贝叶斯优化（Bayesian Optimization）

超参数搜索的主流方法。

| 具体算法 | 代理模型 | 工具 | 特点 |
| --- | --- | --- | --- |
| **GP-BO** | Gaussian Process | scikit-optimize | 经典方法，小搜索空间表现好 |
| **TPE** | Tree-structured Parzen Estimator | **Optuna**, Hyperopt | 高维空间更好，是目前最主流的选择 |
| **SMAC** | Random Forest | SMAC3 | 处理条件超参数能力强 |

**核心原理**：

```text
1. 用代理模型（GP/TPE）拟合"超参数 → 性能"的映射
2. 通过 Acquisition Function（EI/PI/UCB）选择下一组最有希望的参数
3. 评估 → 更新代理模型 → 重复
4. 比 Grid Search / Random Search 少数倍的评估次数
```

#### 2. Warm Start 迁移调参

**核心思想**：不从零搜索，用源域的最优参数作为起点。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **Warm-started BO** | 用源域的超参数评估历史初始化代理模型 | Feurer et al., 2018 (Meta-Learning) |
| **Multi-task BO** | 同时建模多个任务（域）的超参数响应面，共享信息 | Swersky et al., 2013 (NeurIPS) |
| **RGPE (Ranking-weighted GP Ensemble)** | 用多个源任务的 GP 加权组合作为先验 | Feurer et al., 2018 |

**实际操作**：

```text
A省最优参数: {lr: 0.01, depth: 6, n_est: 300}
      ↓
作为B省搜索的起点（Warm Start）
      ↓
Optuna 在 {lr: [0.005, 0.02], depth: [5,7], n_est: [200,400]} 范围内微调
      ↓
通常 10-20 次试验即可收敛（vs 从零搜索需要 50-100 次）
```

#### 3. 多保真度优化（Multi-fidelity）

用少量数据 / 少量 epoch 快速筛选参数，再用全量验证。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **Hyperband** | 基于 Successive Halving，快速淘汰差参数 | Li et al., 2017 (JMLR) |
| **BOHB** | Hyperband + TPE | Falkner et al., 2018 (ICML) |

**适合电力场景的原因**：数据量大（年级别时序数据），全量训练一次很慢，用 10% 数据快速筛参 → 全量验证最优参数，效率提升 5-10 倍。

## 三、模型融合与模型路由

### 3.1 要解决的问题

单一模型难以覆盖所有场景。需要多模型协同。

### 3.2 核心技术方法

#### 1. 集成学习（Ensemble）

| 方法 | 原理 | 适用场景 |
| --- | --- | --- |
| **加权平均** | 每个模型输出乘以权重后求和 | 回归任务，最简单有效 |
| **Stacking** | 用元学习器（meta-learner）学习如何组合基模型输出 | 基模型差异大时效果好 |
| **Bagging** | 训练多个同结构模型（不同数据子集），取平均 | 降低方差 |
| **Boosting** | 序列训练，每个模型修正前一个的残差 | XGBoost/LightGBM 本质就是 |

**加权融合的权重学习**：

```text
方法1：验证集上最小化加权 loss（线性规划 / 网格搜索）
方法2：贝叶斯模型平均（BMA）— 按模型后验概率加权
方法3：Online Learning — 按历史预测精度动态调权
```

#### 2. 场景感知模型路由（Mixture of Experts 思想）

**核心思想**：不是所有模型都参与预测，而是根据当前场景选择最合适的模型。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **规则路由** | 人工定义规则：季节/天气/节假日 → 模型 | 最简单，可解释性强 |
| **Gating Network** | 训练一个"路由网络"，输入当前场景特征，输出各模型的选择权重 | Jacobs et al., 1991 (MoE) |
| **Dynamic Ensemble Selection (DES)** | 根据测试样本的局部邻域，选择在该邻域表现最好的模型 | Cruz et al., 2018 |
| **Multi-armed Bandit** | 在线探索-利用，动态选择表现最好的模型 | Thompson Sampling / UCB |

**电力场景的典型路由策略**：

```text
场景识别（输入特征 → 场景类别）
    ├── 常规工况 → 通用模型（XGBoost）
    ├── 极端高温 → 高温专用模型
    ├── 极端低温 → 低温专用模型
    ├── 节假日 → 节假日模型
    └── 未知场景 → 融合模型（多模型加权）
```

**Gating Network 的简化实现**：

```text
输入: [温度, 湿度, 是否节假日, 季节, 历史负荷趋势]
      ↓
轻量分类器（如小型 MLP / 决策树）
      ↓
输出: softmax([w_model_A, w_model_B, w_model_C])
      ↓
最终预测 = w_A * pred_A + w_B * pred_B + w_C * pred_C
```

#### 3. 大模型 + 小模型协同

这是一个较新的方向：

```text
LLM（场景理解） → 识别当前属于什么场景/工况
      ↓
选择对应的专用预测模型（小模型）
      ↓
返回预测结果 + 场景解释
```

**技术要点**：

- LLM 做的是 NLU 层面的场景分类，不做数值预测
- 小模型做精确数值预测
- LLM 还可以生成预测结果的自然语言解释

**这个方向目前研究较新**，实际落地案例不多，但在电力行业是有价值的：运维人员可以用自然语言查询"明天华东地区负荷怎么样"，LLM 理解意图 → 调用对应模型 → 返回结果 + 解释。

## 四、模型微调与迁移学习

### 4.1 要解决的问题

当特征适配 + 参数调优仍不够时（源域和目标域差异很大），需要用目标域数据微调模型。

### 4.2 核心技术方法

#### 1. Fine-tuning（微调）

最直接的迁移方法。

| 策略 | 做法 | 适用场景 |
| --- | --- | --- |
| **全量微调** | 用目标域数据继续训练所有参数 | 目标域数据充足 |
| **冻结 + 微调** | 冻结底层（特征提取），只微调顶层 | 目标域数据较少（防止过拟合） |
| **判别学习率** | 底层用小学习率，顶层用大学习率 | 平衡保留通用特征和学习新特征 |
| **逐层解冻** | 逐步解冻更多层 | 渐进式适配 |

**对于电力场景的注意点**：

- 树模型（XGBoost）不存在"层"的概念，微调方式是继续 boosting（增加新树）
- 深度学习模型（LSTM/Transformer）才有冻结/解冻的操作空间
- 树模型更常用的是 Warm Start（用源模型作为初始化，继续训练）

#### 2. Domain Adaptation（域适应）

不只是微调，而是显式地对齐源域和目标域的特征分布。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **CORAL** | 对齐源域和目标域特征的二阶统计量（协方差矩阵） | Sun et al., 2016 (AAAI) |
| **MMD (Maximum Mean Discrepancy)** | 最小化两个域在再生核希尔伯特空间中的距离 | Gretton et al., 2012 |
| **DANN (Domain Adversarial NN)** | 对抗训练：特征提取器学习域不变特征，域判别器试图区分域 | Ganin et al., 2016 (JMLR) |
| **Optimal Transport** | 通过最优传输将源域分布"搬运"到目标域 | Courty et al., 2017 |

**电力场景适用性**：

- CORAL 和 MMD 最实用 — 计算简单，对表格数据效果好
- DANN 适合深度模型（LSTM/Transformer 的负荷预测）
- Optimal Transport 理论优美但计算开销大，适合离线场景

**CORAL 的直觉**：

```text
源域特征协方差: C_s
目标域特征协方差: C_t

变换矩阵 A 使得:
  X_s' = X_s * A
  Cov(X_s') ≈ C_t

即：把源域数据的分布"拉"到和目标域一致
然后在变换后的数据上训练/预测
```

#### 3. Few-shot Adaptation（小样本适配）

目标域数据极少时（如新地区刚开始运营）。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **MAML** | 元学习：学习一个好的模型初始化，使得几步梯度下降就能适配新任务 | Finn et al., 2017 (ICML) |
| **Prototypical Networks** | 学习嵌入空间，每个类别一个原型，新样本按距离分类 | Snell et al., 2017 |
| **TrAdaBoost** | 基于 AdaBoost 的迁移方法，自动降低源域中与目标域不一致的样本权重 | Dai et al., 2007 |

**电力场景的实际考虑**：

- Few-shot 在电力场景中出现在：新建变电站、新开发区域、新装光伏电站
- 实操中最常用的是 TrAdaBoost（简单、适合表格数据）和 Warm Start Fine-tune
- MAML 等元学习方法理论好但工程复杂度高

## 五、数据分布漂移监测

### 5.1 要解决的问题

这是**触发机制** — 系统需要知道"什么时候需要适配"。

### 5.2 核心技术方法

#### 1. 数据漂移检测（Data Drift / Covariate Shift）

检测 P(X) 是否发生变化。

| 方法 | 原理 | 特点 |
| --- | --- | --- |
| **PSI (Population Stability Index)** | 比较两个分布的分箱频率差异 | 工业界最常用，简单直观，有明确阈值（<0.1 稳定，0.1-0.25 需关注，>0.25 显著漂移） |
| **KS test (Kolmogorov-Smirnov)** | 比较两个分布的 CDF 最大差异 | 非参数检验，单变量，有 p-value |
| **KL divergence** | 信息论度量，衡量两个分布的差异 | 不对称，常用 Jensen-Shannon 对称版本 |
| **Wasserstein Distance** | 最优传输距离（"推土机距离"） | 数值稳定，对分布形状变化敏感 |
| **MMD** | 核方法度量分布差异 | 适合多维联合分布检测 |

**多变量 vs 单变量**：

```text
单变量检测: 对每个特征分别做 KS/PSI 检测
  - 优点：简单，可定位哪个特征漂移
  - 缺点：忽略特征间相关性变化

多变量检测: 对联合分布做 MMD / 多变量 KS
  - 优点：捕获整体分布变化
  - 缺点：无法定位具体特征

实际建议: 两者结合 — PSI 逐特征 + MMD 整体
```

#### 2. 概念漂移检测（Concept Drift）

检测 P(Y|X) 是否发生变化 — 即使输入分布没变，输入到输出的关系变了。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **DDM (Drift Detection Method)** | 监控模型错误率的均值和标准差，超出阈值报警 | Gama et al., 2004 |
| **EDDM (Early DDM)** | DDM 改进，对缓慢漂移更敏感 | Baena-García et al., 2006 |
| **ADWIN (Adaptive Windowing)** | 自适应窗口，自动检测均值变化的最优切割点 | Bifet & Gavaldà, 2007 |
| **Page-Hinkley test** | 序贯分析，检测均值的持续偏移 | Page, 1954 |
| **CUSUM** | 累积和控制图，检测过程均值的微小持续偏移 | 经典统计过程控制方法 |

**电力场景的概念漂移来源**：

- 产业结构变化（新工厂投产/关停）
- 居民用电习惯变化（电动车普及）
- 电网改造（新变电站投运）
- 气候长期趋势（年际气温变化）

#### 3. 漂移 → 适配的触发策略

```text
漂移检测结果
    ├── 无漂移 → 继续使用当前模型
    ├── 轻度漂移（PSI 0.1-0.25）→ 触发参数微调
    ├── 中度漂移（PSI 0.25-0.5）→ 触发特征重选 + 参数调优
    └── 重度漂移（PSI > 0.5）→ 触发模型微调 / 重新训练
```

这形成了一个 **分级响应机制**：

```text
成本:    低 ──────────────────→ 高
响应:    参数调优 → 特征重选 → 模型微调 → 重新训练
触发:    轻度漂移   中度漂移    重度漂移    根本性变化
```

## 六、视觉模型适配技术

### 6.1 要解决的问题

视觉类模型（图像分类、目标检测、缺陷识别等）的跨域适配与表格数据有本质区别：特征空间是像素/卷积特征，分布偏移往往表现为光照、视角、背景、设备差异。

### 6.2 核心技术方法

#### 1. 视觉特征适配

目的：理解和调整视觉模型关注的特征区域。

| 方法 | 原理 | 适用场景 | 参考 |
| --- | --- | --- | --- |
| **Grad-CAM** | 利用梯度加权最后一层卷积特征图，可视化模型关注区域 | 诊断模型是否关注正确的视觉特征 | Selvaraju et al., 2017 (ICCV) |
| **Attention Rollout** | 聚合 Vision Transformer 多层 Attention 权重 | ViT 架构的特征可视化 | Abnar & Zuidema, 2020 |
| **Feature Visualization** | 通过优化输入图像最大化某层激活，可视化学到的特征 | 理解每层学到了什么 | Olah et al., 2017 (Distill) |

**电力场景应用**：

- 设备缺陷检测：用 Grad-CAM 验证模型是否关注缺陷区域（而非背景）
- 跨区域适配时，对比源域和目标域的 Grad-CAM 热力图，判断特征迁移是否合理

#### 2. 视觉域适应（Visual Domain Adaptation）

目的：对齐源域和目标域的视觉特征分布，使模型泛化到新的视觉环境。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **DANN (Domain Adversarial NN)** | 特征提取器 + 域判别器对抗训练，学习域不变视觉特征 | Ganin et al., 2016 (JMLR) |
| **ADDA (Adversarial Discriminative DA)** | 先训练源域编码器，再对抗训练目标域编码器对齐分布 | Tzeng et al., 2017 (CVPR) |
| **Deep CORAL** | 最小化源域和目标域深层特征的协方差矩阵差异 | Sun & Saenko, 2016 (ECCV Workshop) |
| **BN Adaptation** | 用目标域数据更新 BatchNorm 层的均值和方差统计量 | Li et al., 2018 |
| **SHOT** | 仅需目标域数据（无标签），通过信息最大化 + 伪标签自适应 | Liang et al., 2020 (ICML) |
| **MCD (Maximum Classifier Discrepancy)** | 两个分类器最大化差异 + 特征提取器最小化差异 | Saito et al., 2018 (CVPR) |

**电力场景适用性**：

- **BN Adaptation** — 最简单，仅需目标域无标签图像前向传播一遍即可，适合快速试验
- **SHOT** — 无需源域数据，适合模型共享后在目标端独立适配
- **DANN / ADDA** — 需要同时访问源域和目标域数据，效果好但部署更复杂
- **Deep CORAL** — 在 CNN 中间层加正则项，实现简单，效果稳定

**BN Adaptation 的直觉**：

```text
预训练模型的 BN 层保存了源域的 (mean, var)
       ↓
在目标域图像上做若干轮前向传播
       ↓
BN 层的 running_mean 和 running_var 更新为目标域统计量
       ↓
模型即适配了目标域的光照/色彩/对比度分布
成本极低，无需反向传播，无需标签
```

#### 3. 视觉迁移学习

目的：利用预训练视觉模型快速适配到目标任务。

| 策略 | 做法 | 适用场景 |
| --- | --- | --- |
| **冻结骨干 + 微调头** | 保留 ResNet/ViT 的特征提取层，只训练分类/检测头 | 目标域数据少（几百张） |
| **逐层解冻** | 先训练头，再逐步解冻更深的层 | 渐进式适配，防止灾难性遗忘 |
| **LoRA (Low-Rank Adaptation)** | 在 Transformer 层注入低秩矩阵，仅训练这些参数 | ViT 架构，极少参数量（<1% 原模型） |
| **Adapter Modules** | 在每层之间插入小型瓶颈模块，仅训练 Adapter | 多任务/多域共享同一骨干 |
| **Visual Prompt Tuning (VPT)** | 在输入序列中添加可学习的 token | ViT 架构，参数效率极高 |

**参数效率对比**：

```text
方法              可训练参数量     适合数据量
全量微调          100%            > 10,000 张
冻结骨干+微调头    < 5%           100-1,000 张
LoRA             < 1%            100-1,000 张
VPT              < 0.1%          50-500 张
Adapter          ~ 2-5%          500-2,000 张
```

**电力场景的注意点**：

- 电力设备图像（变压器、绝缘子、输电线路）与 ImageNet 预训练差异较大
- 推荐：先在电力设备大数据集上做领域预训练（domain-specific pretraining），再做下游任务微调
- LoRA / Adapter 适合多区域共享同一骨干，每个区域保存自己的轻量适配参数

#### 4. 视觉数据增强与合成

目的：通过数据层面的手段弥补目标域数据不足或分布差异。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **CycleGAN** | 无配对数据的图像风格迁移，将源域图像转换为目标域风格 | Zhu et al., 2017 (ICCV) |
| **Domain Randomization** | 训练时随机化光照/背景/纹理，使模型对域变化鲁棒 | Tobin et al., 2017 |
| **MixStyle** | 在 CNN 特征空间中混合不同样本的统计量（风格） | Zhou et al., 2021 (ICLR) |
| **RandAugment** | 随机组合多种增强策略，简化增强超参数搜索 | Cubuk et al., 2020 (NeurIPS) |
| **Copy-Paste Augmentation** | 将缺陷区域复制粘贴到正常图像上生成训练样本 | Ghiasi et al., 2021 (CVPR) |

**电力场景应用**：

```text
场景1 — 跨设备型号适配:
  CycleGAN: 将A型号设备图像 → B型号风格
  训练时使用混合数据，提升泛化能力

场景2 — 缺陷样本不足:
  Copy-Paste: 从已有缺陷图像中裁剪缺陷区域 → 粘贴到正常图像
  快速扩充稀缺类别的训练数据

场景3 — 多现场部署:
  Domain Randomization: 随机化光照/角度
  MixStyle: 混合不同现场的特征统计量
  使模型对不同现场环境鲁棒
```

#### 5. 视觉漂移检测

目的：检测视觉数据分布是否发生变化，触发模型更新。

| 方法 | 原理 | 参考 |
| --- | --- | --- |
| **FID (Frechet Inception Distance)** | 比较两组图像在 Inception 特征空间中的分布距离 | Heusel et al., 2017 |
| **MMD on CNN Features** | 在 CNN 中间层特征上计算 MMD | Gretton et al., 2012 |
| **LPIPS** | 用深度网络提取感知特征，计算图像间的感知距离 | Zhang et al., 2018 (CVPR) |
| **Autoencoder 重建误差** | 用源域训练 AE，目标域重建误差升高即表示漂移 | 经典异常检测思路 |

**电力场景的视觉漂移来源**：

- 摄像头老化/更换（色彩偏移、分辨率变化）
- 季节变化（光照角度、植被遮挡）
- 设备维护（清洗前后外观差异）
- 新增设备型号（未见过的形态）

**检测流程**：

```text
定期采集目标域新图像
       ↓
提取 CNN 特征（用预训练骨干）
       ↓
与基准分布计算 FID / MMD
       ↓
FID 超阈值 → 轻度: BN Adaptation
             中度: LoRA / Adapter 微调
             重度: 全量微调 / 重新训练
```
